# /remember — 记忆管理

显式管理会话级的关键上下文信息。帮助 Agent 在长对话中保持对重要决策、约束、偏好和临时笔记的记忆，避免在后续操作中遗忘或违背用户的关键指示。

## 触发词
- "记住"
- "别忘了"
- "remember"
- "之前说的还记得吗"
- "我说过的约束"
- "记一下"

## 执行流程

### 第一步：解析记忆意图（工具：无，纯推理）

分析用户输入，判断要执行的记忆操作类型：

| 操作 | 触发特征 | 示例 |
|------|----------|------|
| **存储** | "记住…"、"别忘了…"、"从现在起…" | "记住：数据库字段用下划线命名" |
| **检索** | "之前说的…"、"还记得…吗"、"我说过…" | "之前说的技术栈约束还记得吗？" |
| **列出** | "列出所有记忆"、"记了什么"、"看看笔记" | "帮我看看我之前让你记住了什么" |
| **清除** | "忘掉…"、"取消那个约束"、"不再需要…" | "忘掉之前说的不能改QueryEngine" |
| **更新** | "改一下…"、"更正…"、"其实应该是…" | "之前说用Java 17，改成Java 21" |

提取记忆内容的关键要素：
- **记忆主体**：具体要记住什么
- **记忆分类**：决策 / 约束 / 偏好 / 上下文 / 临时笔记
- **作用范围**：全局适用 / 仅限特定模块 / 仅限当前任务

> **失败处理**：用户表达模糊时，复述理解内容并向用户确认，如"你是想让我记住'数据库字段用下划线风格'这个约束吗？"

### 第二步：分类标记（工具：无，纯推理）

将记忆内容归入以下分类体系：

```
📂 记忆分类体系
├── 📌 决策 — 已做出的技术决策和选择
│   例："使用 PostgreSQL 而非 MySQL"
│   例："API 使用 RESTful 风格"
├── 🚫 约束 — 不可违背的限制条件
│   例："不修改 QueryEngine 核心循环"
│   例："不使用 Lombok 的 @Data 注解"
├── 💡 偏好 — 用户的风格和习惯偏好
│   例："数据库字段用下划线命名"
│   例："代码注释用中文"
├── 📋 上下文 — 当前任务的背景信息
│   例："正在做用户模块重构"
│   例："这个 bug 跟缓存失效有关"
└── 📝 临时笔记 — 短期有用的备忘信息
    例："待验证：OrderService 的事务边界"
    例："明天需要更新 API 文档"
```

为每条记忆生成：
- 分类标签
- 时间戳
- 一行摘要

> **失败处理**：无法明确分类时，默认归入"上下文"类别，后续可由用户调整。

### 第三步：存储执行（工具：FileRead / FileWrite）

将记忆持久化到会话记忆文件：

1. **读取现有记忆文件**（如存在）
   ```bash
   # 记忆文件路径
   .zhikun/scratchpad/session-memory.md
   ```
2. **执行相应操作**：
   - **存储**：在对应分类下追加新条目
   - **检索**：搜索并返回匹配的记忆条目
   - **列出**：返回所有分类下的记忆清单
   - **清除**：从文件中删除指定条目
   - **更新**：找到旧条目，替换为新内容并标注更新时间
3. **写入记忆文件**

记忆文件格式：

```markdown
# Session Memory

> 最后更新：{timestamp}

## 📌 决策
- [{date}] 项目使用 Java 21 语法标准
- [{date}] API 认证使用 JWT + Redis 方案

## 🚫 约束
- [{date}] 不修改 QueryEngine 核心循环结构
- [{date}] 不在 Controller 层写业务逻辑

## 💡 偏好
- [{date}] 数据库字段用下划线命名风格
- [{date}] 代码注释使用中文

## 📋 上下文
- [{date}] 当前任务：用户模块重构，涉及 UserService 和 UserController
- [{date}] 已知问题：缓存在高并发下偶尔失效

## 📝 临时笔记
- [{date}] 待验证：OrderService 的事务边界是否正确
- [{date}] PaymentService 的回调地址需要更换
```

> **失败处理**：
> - 记忆文件不存在 → 自动创建新文件并初始化模板
> - 文件写入失败 → 将记忆内容直接输出给用户，建议手动保存
> - 存储时发现重复 → 提示用户已有类似记忆，询问是否更新

### 第四步：确认反馈（工具：无，纯输出）

操作完成后，给用户清晰的反馈：

**存储成功**：
```
✅ 已记住（约束）："不修改 QueryEngine 核心循环结构"
📁 存储位置：.zhikun/scratchpad/session-memory.md
💡 后续执行代码修改时，我会自动遵守这个约束。
```

**检索结果**：
```
🔍 找到 {N} 条相关记忆：

📌 决策：
  - [2026-05-16] 项目使用 Java 21 语法标准

🚫 约束：
  - [2026-05-16] 不修改 QueryEngine 核心循环结构
```

**列出全部**：
```
📋 当前记忆清单（共 {N} 条）：
  📌 决策：{M} 条
  🚫 约束：{M} 条
  💡 偏好：{M} 条
  📋 上下文：{M} 条
  📝 临时笔记：{M} 条
```

**清除确认**：
```
🗑️ 已清除约束："不修改 QueryEngine 核心循环结构"
⚠️ 后续操作将不再受此约束限制。
```

**更新确认**：
```
🔄 已更新决策：
  旧："项目使用 Java 17"
  新："项目使用 Java 21"
```

```
### 操作详情
{具体的操作结果描述}

- 📌 决策：{count} 条

- 💡 偏好：{count} 条
- 📋 上下文：{count} 条

{说明这条记忆如何在后续对话中生效}
```

## 原则
- **显式优先**：只存储用户明确要求记住的内容，不自作主张推断和存储
- **约束至上**：约束类记忆具有最高优先级，在后续任何操作中都必须遵守
- **及时清理**：任务完成后提醒用户清理临时笔记，保持记忆文件精简
- **冲突检测**：存储新记忆时检查是否与已有记忆冲突，冲突时提示用户决定
- **记忆可溯**：每条记忆带时间戳和分类，支持快速定位和回顾
