### P0-03/04: FileReadTool / FileEditTool 路径遍历修复

#### Claude Code 原版实现分析

**Claude Code 的 FileReadTool** (`FileReadTool.ts` L398-405) 通过统一的权限系统进行路径验证：


`checkReadPermissionForTool` 最终调用 `pathInAllowedWorkingPath()`（在 `filesystem.ts` 1778行中），核心检查：
1. **deny 规则优先检查** — 命中 deny 规则直接拒绝
2. **工作目录边界检查** — 路径必须在 `workingDirectories` 内
3. **危险文件保护** — `.gitconfig`, `.bashrc` 等即使在工作目录也受保护
4. **符号链接解析** — 先 `safeResolvePath()` 再检查

**Claude Code 的 FileEditTool** (`FileEditTool.ts` L137-361) 还增加了：
- **readFileState 校验** — 文件未被 Read 过则拒绝编辑（L276-287）
- **文件修改时间校验** — `getFileModificationTime()` 检测第三方修改（L290-311）
- **Secret 检查** — `checkTeamMemSecrets()` 阻止写入 secret（L144-147）
- **Settings 文件保护** — `validateInputForSettingsFileEdit()`（L346-358）

#### ZhikuCode 修复方案

**Step 1: 创建统一的路径安全验证服务**

```java
// 新建: backend/src/main/java/com/aicodeassistant/security/PathSecurityService.java

@Service
public class PathSecurityService {

    private static final Logger log = LoggerFactory.getLogger(PathSecurityService.class);

    /** 危险文件 — 即使在项目目录内也需要 ask 确认 */
    private static final Set<String> DANGEROUS_FILES = Set.of(
        ".gitconfig", ".gitmodules", ".bashrc", ".bash_profile",
        ".zshrc", ".zprofile", ".profile", ".env", ".env.local"
    );

    /** 危险目录 — 自动编辑时需要额外确认 */
    private static final Set<String> DANGEROUS_DIRECTORIES = Set.of(
        ".git", ".vscode", ".idea", ".claude", ".ssh", ".gnupg", ".aws"
    );

    /** 设备文件黑名单 */
    private static final Set<String> BLOCKED_DEVICE_PATHS = Set.of(
        "/dev/zero", "/dev/random", "/dev/urandom", "/dev/full",
        "/dev/stdin", "/dev/tty", "/dev/console",
        "/dev/stdout", "/dev/stderr",
        "/dev/fd/0", "/dev/fd/1", "/dev/fd/2"
    );

    /**
     * 验证读取路径安全性
     * @param filePath 原始文件路径
     * @param workingDirectory 项目工作目录
     * @return 验证结果: allowed=true 或 错误消息
     */
    public PathCheckResult checkReadPermission(String filePath, String workingDirectory) {
        Path resolved = resolvePath(filePath, workingDirectory);
        
        // 1. 设备文件检查
        if (BLOCKED_DEVICE_PATHS.contains(resolved.toString())) {
            return PathCheckResult.denied("Cannot read device file: " + resolved);
        }
        
        // 2. /proc 特殊文件检查
        String pathStr = resolved.toString();
        if (pathStr.startsWith("/proc/") && 
            (pathStr.endsWith("/fd/0") || pathStr.endsWith("/fd/1") || pathStr.endsWith("/fd/2"))) {
            return PathCheckResult.denied("Cannot read process fd file");
        }

        // 3. 项目边界检查 — 对齐 Claude Code pathInAllowedWorkingPath()
        Path projectRoot = Path.of(workingDirectory).toAbsolutePath().normalize();
        if (!resolved.startsWith(projectRoot)) {
            // 允许的外部路径白名单
            if (!isAllowedExternalPath(resolved)) {
                return PathCheckResult.denied(
                    "Access denied: path '" + filePath + "' is outside project boundary. " +
                    "Allowed: " + projectRoot);
            }
        }

        // 4. 危险文件警告（不阻止，但标记需要确认）
        String fileName = resolved.getFileName().toString().toLowerCase();
        if (DANGEROUS_FILES.contains(fileName)) {
            return PathCheckResult.needsConfirmation(
                "Reading sensitive file: " + fileName);
        }

        return PathCheckResult.allowed();
    }

    /**
     * 验证写入路径安全性 — 比读取更严格
     */
    public PathCheckResult checkWritePermission(String filePath, String workingDirectory) {
        // 先检查读取权限（包含所有基础检查）
        PathCheckResult readCheck = checkReadPermission(filePath, workingDirectory);
        if (!readCheck.isAllowed() && !readCheck.needsConfirmation()) {
            return readCheck;
        }

        Path resolved = resolvePath(filePath, workingDirectory);

        // 5. 危险目录写入检查
        for (String dangerDir : DANGEROUS_DIRECTORIES) {
            if (containsPathComponent(resolved, dangerDir)) {
                return PathCheckResult.needsConfirmation(
                    "Writing to protected directory: " + dangerDir);
            }
        }

        return readCheck;
    }

    /** 解析路径 — 处理相对路径、符号链接、../ 遍历 */
    public Path resolvePath(String filePath, String workingDirectory) {
        Path path = Path.of(filePath);
        Path resolved;
        if (path.isAbsolute()) {
            resolved = path;
        } else {
            resolved = Path.of(workingDirectory).resolve(filePath);
        }
        // normalize() 消除 ../ 遍历
        resolved = resolved.toAbsolutePath().normalize();
        
        // 符号链接解析（可选，对齐 Claude Code safeResolvePath）
        try {
            resolved = resolved.toRealPath();
        } catch (IOException e) {
            // 文件不存在时 toRealPath 会失败，使用 normalize 结果
        }
        return resolved;
    }

    private boolean isAllowedExternalPath(Path path) {
        String pathStr = path.toString();
        return pathStr.startsWith("/tmp") ||
               pathStr.startsWith(System.getProperty("java.io.tmpdir"));
    }

    private boolean containsPathComponent(Path path, String component) {
        for (Path part : path) {
            if (part.toString().equalsIgnoreCase(component)) return true;
        }
        return false;
    }

    /** 路径检查结果 */
    public record PathCheckResult(boolean isAllowed, boolean needsConfirmation, String message) {
        public static PathCheckResult allowed() { return new PathCheckResult(true, false, null); }
        public static PathCheckResult denied(String msg) { return new PathCheckResult(false, false, msg); }
        public static PathCheckResult needsConfirmation(String msg) { return new PathCheckResult(true, true, msg); }
    }
}
```

### P0-05: WebFetchTool SSRF 修复

#### Claude Code 原版实现分析

**Claude Code 的 WebFetchTool** 采用多层防御：

1. **域名预审批白名单** (`preapproved.ts` 167行) — 131个预审批的代码相关域名
2. **域名级权限规则** (`WebFetchTool.ts` L104-180) — allow/deny/ask 按 `domain:hostname` 粒度
3. **URL 缓存** (`utils.ts` L62-69) — LRU 缓存 15min TTL + 50MB 上限

**关键**: Claude Code 没有直接做 SSRF IP 检查（因为权限系统已经限制了域名），但 ZhikuCode 没有权限系统，必须在网络层防护。

#### ZhikuCode 修复方案

```java
// WebFetchTool.java — 添加 URL 安全验证

private static final Set<String> BLOCKED_SCHEMES = Set.of("file", "ftp", "gopher", "data");
private static final Set<String> PREAPPROVED_HOSTS = Set.of(
    // 对齐 Claude Code preapproved.ts — 精选代码相关域名
    "docs.python.org", "docs.oracle.com", "developer.mozilla.org",
    "docs.spring.io", "react.dev", "vuejs.org", "nextjs.org",
    "www.postgresql.org", "redis.io", "kubernetes.io",
    "git-scm.com", "github.com", "stackoverflow.com"
);

/**
 * URL 安全验证 — 防止 SSRF 攻击
 * 对齐 Claude Code: preapproved 域名免确认 + 其他域名需权限检查
 */
private ToolResult validateUrl(String urlStr) {
    try {
        URI uri = new URI(urlStr);
        
        // 1. 协议检查 — 仅允许 http/https
        String scheme = uri.getScheme();
        if (scheme == null || BLOCKED_SCHEMES.contains(scheme.toLowerCase())) {
            return ToolResult.error("Blocked URL scheme: " + scheme + ". Only http/https allowed.");
        }
        if (!"http".equalsIgnoreCase(scheme) && !"https".equalsIgnoreCase(scheme)) {
            return ToolResult.error("Unsupported URL scheme: " + scheme);
        }
        
        // 2. Host 检查
        String host = uri.getHost();
        if (host == null || host.isBlank()) {
            return ToolResult.error("Invalid URL: no host specified");
        }
        
        // 3. IP 地址解析 — 阻止内网/环回/链路本地
        InetAddress addr = InetAddress.getByName(host);
        if (addr.isLoopbackAddress()) {
            return ToolResult.error("SSRF blocked: loopback address");
        }
        if (addr.isSiteLocalAddress()) {
            return ToolResult.error("SSRF blocked: private network address");
        }
        if (addr.isLinkLocalAddress()) {
            return ToolResult.error("SSRF blocked: link-local address");
        }
        // AWS/GCP/Azure 元数据服务
        if ("169.254.169.254".equals(addr.getHostAddress())) {
            return ToolResult.error("SSRF blocked: cloud metadata service");
        }
        
        return null; // 验证通过
    } catch (Exception e) {
        return ToolResult.error("Invalid URL: " + e.getMessage());
    }
}

// 修改 call() 方法
@Override
public ToolResult call(ToolInput input, ToolUseContext context) {
    String url = input.getString("url");
    
    // ★ 新增: SSRF 防护
    ToolResult urlCheck = validateUrl(url);
    if (urlCheck != null) return urlCheck;
    
    // ★ 修复: 流式大小检查替代全量读入
    // ...
}
```

### P0-08: WebSocket STOMP 认证修复

#### Claude Code 对比

Claude Code 没有 WebSocket 层（使用 Ink terminal UI），不适用。但 ZhikuCode 的 WebSocket 层必须有认证。

#### 修复方案

```java
// WebSocketConfig.java — 将 StompAuthInterceptor 改为非 static + 注入 RemoteAccessSecurityFilter

@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    private final RemoteAccessSecurityFilter remoteAccessFilter;

    public WebSocketConfig(RemoteAccessSecurityFilter remoteAccessFilter) {
        this.remoteAccessFilter = remoteAccessFilter;
    }

    @Override
    public void configureClientInboundChannel(ChannelRegistration registration) {
        registration.interceptors(new StompAuthInterceptor(remoteAccessFilter));
    }

    /** STOMP 认证拦截器 — 验证 Bearer Token */
    private static class StompAuthInterceptor implements ChannelInterceptor {
        private final RemoteAccessSecurityFilter remoteAccessFilter;
        
        StompAuthInterceptor(RemoteAccessSecurityFilter filter) {
            this.remoteAccessFilter = filter;
        }

        @Override
        public Message<?> preSend(Message<?> message, MessageChannel channel) {
            StompHeaderAccessor accessor = MessageHeaderAccessor
                    .getAccessor(message, StompHeaderAccessor.class);

            if (accessor != null && StompCommand.CONNECT.equals(accessor.getCommand())) {
                String authHeader = accessor.getFirstNativeHeader("Authorization");
                
                if (authHeader == null || !authHeader.startsWith("Bearer ")) {
                    throw new MessageDeliveryException("Authentication required: Bearer token missing");
                }
                
                String token = authHeader.substring(7);
                // ★ 修复: 验证 token 与 RemoteAccessSecurityFilter 的 accessToken 匹配
                if (!remoteAccessFilter.getAccessToken().equals(token)) {
                    throw new MessageDeliveryException("Authentication failed: invalid access token");
                }
                
                String principalName = "user-" + token.substring(Math.max(0, token.length() - 8));
                accessor.setUser(new StompPrincipal(principalName));
                
                // 提取 sessionId
                String sessionId = accessor.getFirstNativeHeader("X-Session-Id");
                if (sessionId != null) {
                    Map<String, Object> attrs = accessor.getSessionAttributes();
                    if (attrs == null) {
                        attrs = new ConcurrentHashMap<>();
                        accessor.setSessionAttributes(attrs);
                    }
                    attrs.put("sessionId", sessionId);
                }
            }
            return message;
        }
    }
}
```

## 四、架构对齐指导

### 4.1 Claude Code 架构模式 vs ZhikuCode 当前差异

| 维度 | Claude Code 模式 | ZhikuCode 当前 | 对齐建议 |
|------|------------------|----------------|----------|
| **权限检查入口** | 每个 Tool 有 `checkPermissions()` 回调，由 QueryEngine 统一调度 | PermissionPipeline 集中式 7 步管线 | 保持管线架构，但在每个 Tool 增加 `checkPermissions()` 预检 |
| **路径验证** | `utils/permissions/pathValidation.ts` (486行) 统一服务 | PathValidator 仅 BashTool 使用 | ★ 创建 `PathSecurityService`，FileRead/Edit/Bash 共用 |

### P2-03/04: 前端类型安全

**Claude Code 对比**: Claude Code 前端使用 Ink (React for CLI)，消息类型通过 Zod schema 严格定义。

**修复**: 替换 `dispatch.ts` 中的 `any` 和 `as never`:

```typescript
// dispatch.ts — 使用 discriminated union
type MessageHandler<T extends ServerMessage['type']> = 
    (data: Extract<ServerMessage, { type: T }>) => void;

const handlers: { [K in ServerMessage['type']]?: MessageHandler<K> } = {
    'stream_delta': (d) => { /* d 已自动推导为 StreamDelta 类型 */ },
    // ...
};

export function dispatch(data: ServerMessage & { ts?: number }): void {
    const handler = handlers[data.type];
    if (handler) {
        (handler as (d: ServerMessage) => void)(data);
    }
}
```
