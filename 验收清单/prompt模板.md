## [Hard Constraints] 硬性约束

1. **技术规范强制对齐**：所有评估必须基于 Java 21 + Spring Boot 3.x(非WebFlux) + Python 3.11 + FastAPI + React 18 + Vite，不允许假设使用文档未指定的技术
2. **P0/P1边界严格**：只审查P0和P1功能，P2功能明确标注跳过，不纳入评估
3. **前端交互深度审查**：终端UI(Ink)到Web UI的转换是最大风险区域——每个P0/P1前端功能必须逐一验证在浏览器中的可实现性，特别是：键盘快捷键、流式渲染、虚拟滚动、权限对话框、命令面板、Monaco编辑器集成
4. **跨栈通信审查**：原版进程内同步调用变为HTTP/WebSocket异步通信，必须逐一评估延迟、状态同步、断连恢复等影响
5. **"可实现"的定义**：必须有明确的技术路径（具体到使用哪个库、哪个API），不能含糊说"可以做到"
