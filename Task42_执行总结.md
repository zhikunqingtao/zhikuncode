## 关键洞察

### 1. 第十三章（单元测试）vs 第十四章（端到端）的差异

| 维度 | 第十三章发现 | 第十四章发现 | 差异说明 |
|------|-----------|-----------|---------|
| Agent Loop | 工具摘要/newContext 缺失 | ✅ 通过 | 单元测试场景无法触发长输出 |
| 权限系统 | AutoModeClassifier 桩 | ✅ 通过 | REST 使用 BYPASS 模式规避了 AUTO 路径 |
| 工具系统 | Schema 验证骨架 | ✅ 通过 | LLM 生成的参数恰好合法 |
| 多 Agent | 框架完整 | ❌ 子代理失败 | 模型映射问题在运行时暴露 |
| 上下文管理 | 5 主层完整 | ⚠️ REST 断裂 | REST 路径消息未持久化是新发现 |

**结论**：单元测试和端到端测试各有优势，单元测试适合发现设计缺陷，端到端测试适合发现集成问题。

### 2. 测试覆盖的盲区

第十三章未发现但第十四章发现的问题：
- REST API 消息未持久化（跨请求上下文）
- ToolUseContext NPE（多控制器路径）
- 子代理模型映射失败（运行时环境依赖）
- 自动压缩阈值异常（真实消息量触发）

**启示**：需要补充集成测试用例，覆盖跨模块的真实场景。

### 3. 硬编码问题的广泛性

多个模块存在硬编码问题：
- P-AGENT-01：子代理默认模型硬编码 "haiku"
- P-AGENT-03：AgentTool 模型枚举硬编码
- 涉及文件：AgentDefinition.java、AgentTool.java、AutoModeClassifier.java

**建议**：统一建立配置管理体系，将所有硬编码值迁移到配置文件。

### 4. 测试基础设施不完整

三类测试失败：
1. **构造器签名变更未同步**（7 个测试文件）— 需要自动化工具同步构造器签名
2. **Mock 不完整**（19 + 45 个测试）— 需要改进测试基础设施和 Mock 标准
3. **环境初始化缺失**（BashParserGoldenTest）— 需要统一的测试初始化框架
