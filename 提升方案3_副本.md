## 二、安全阻断项 — P0 紧急修复

### 2.1 ToolExecutionPipeline 权限空洞 📘

**现状**: `backend/.../tool/ToolExecutionPipeline.java` (139 行) 第 94-98 行:

```java
// ToolExecutionPipeline.java 第 94-98 行 — 安全空洞
// ── 阶段 4: 权限检查 ──
PermissionRequirement permReq = tool.getPermissionRequirement();
if (permReq == PermissionRequirement.ALWAYS_ASK) {
    log.debug("Tool {} requires permission (always_ask), auto-allowing in P0", toolName);
}
// ↑ ALWAYS_ASK 工具 (FileWrite/BashTool 写入/FileEditTool) 被直接放行!
```

**影响范围**: 所有标记 `ALWAYS_ASK` 的工具（BashTool 写入、FileWriteTool、FileEditTool、NotebookEditTool）**无需任何用户确认即可执行**。

**修复方案** (替换第 94-98 行):

> ⚠️ **v2.2 修正**: PermissionPipeline 已实现完整的 7 步权限检查 (`checkPermission()` 方法, L68-151)，包含 deny 规则→ask 规则→工具自身检查→工具拒绝→用户交互→安全检查→模式转换。修复代码应调用已有 API `checkPermission()`，而非不存在的 `classify()`/`matchRules()`。

```java
// ── 阶段 4: 权限检查 ──
PermissionRequirement permReq = tool.getPermissionRequirement();
if (permReq != PermissionRequirement.NONE) {
    // 构建权限上下文
    PermissionContext permContext = PermissionContext.of(
        context.permissionMode(), permissionRuleRepository.findAll());
    
    // 调用已有的 7 步权限管线
    PermissionDecision decision = permissionPipeline.checkPermission(
        tool, processedInput, context, permContext);
    
    switch (decision.behavior()) {
        case DENY -> {
            return ToolResult.error("Permission denied: " + decision.reason());
        }
        case ASK -> {
            // 推送到前端请求用户确认（需实现 requestPermission/resolvePermission）
            PermissionDecision userDecision = permissionPipeline.requestAndWaitPermission(
                tool.getName(), processedInput, context);
            if (userDecision.behavior() != PermissionBehavior.ALLOW) {
                return ToolResult.error("Permission denied by user");
            }
        }
        case ALLOW, PASSTHROUGH -> {
            // 允许通过
        }
    }
}
```

> 📝 **关键缺失**: PermissionPipeline 目前缺少 `requestAndWaitPermission()` 异步阻塞等待方法（类似 ElicitationService 的 CompletableFuture 模式），需新增。

**工作量**: 3-5 天 | **优先级**: 🔴 P0

@ConditionalOnProperty(name = "llm.provider", havingValue = "anthropic")
public class AnthropicProvider implements LlmProvider {

    // 1. 请求构建 — system 独立字段 + cache_control
    private ObjectNode buildRequest(String model, List<MessageParam> messages,
                                     String systemPrompt, List<ToolDefinition> tools,
                                     int maxTokens, ThinkingConfig thinking) {
        ObjectNode root = objectMapper.createObjectNode();
        root.put("model", model);
        root.put("max_tokens", maxTokens);
        root.put("stream", true);

        ArrayNode systemArray = root.putArray("system");
        ObjectNode sysPart = systemArray.addObject();
        sysPart.put("type", "text");
        sysPart.put("text", systemPrompt);
        sysPart.putObject("cache_control").put("type", "ephemeral");

        if (thinking instanceof ThinkingConfig.Enabled enabled) {
            ObjectNode thinkNode = root.putObject("thinking");
            thinkNode.put("type", "enabled");
            thinkNode.put("budget_tokens", enabled.budgetTokens());
        }
        return root;
    }

    // 2. SSE 解析 — Anthropic 特有事件类型
    // event: message_start     → 初始化

### 4.4 StreamingToolExecutor 并发安全修复 📘

**现状**: `backend/.../tool/StreamingToolExecutor.java` (191 行) 第 104-133 行 `processQueue()` 存在并发竞态：

```java
// 竞态: peek() 和 poll() 之间非原子
private void processQueue() {
    while (!queue.isEmpty()) {
        TrackedTool next = queue.peek();   // 非原子读取
        if (!canExecute(next)) break;
        queue.poll();                      // 非原子移除
        active.incrementAndGet();
        Thread.startVirtualThread(() -> {
            try { ... } finally {
                active.decrementAndGet();
                processQueue();            // 递归导致多线程同时执行
            }
        });
    }
}
```

**竞态场景**: VirtualThread A 和 B 同时 `processQueue()` → 都 `peek()` 到同一元素 → 双重执行

**修复**:

```java
private final Object queueLock = new Object();

private void processQueue() {
    synchronized (queueLock) {
        while (!queue.isEmpty()) {
            TrackedTool next = queue.peek();
            if (!canExecute(next)) break;
            queue.poll();
            active.incrementAndGet();
            next.state = ToolState.EXECUTING;
            Thread.startVirtualThread(() -> {
                try { /* 工具执行 */ } finally {
                    active.decrementAndGet();
                    processQueue();  // synchronized 防止并发进入
                }
            });
        }
    }
}
```

1. 工具级超时: `Tool.getMaxExecutionTimeMs()` 默认 30s，超时 SIGTERM → 2s → SIGKILL

@Component
public class SensitiveDataFilter {
    private static final List<Pattern> PATTERNS = List.of(
        Pattern.compile("(sk-[a-zA-Z0-9]{20,})"),                    // OpenAI
        Pattern.compile("(AKIA[0-9A-Z]{16})"),                       // AWS
        Pattern.compile("(ghp_[a-zA-Z0-9]{36})"),                    // GitHub
        Pattern.compile("(?i)(api[_-]?key|secret|password|token|auth)\\s*[=:]\\s*['\"]?([^\\s'\"]{8,})"),
        Pattern.compile("(eyJ[a-zA-Z0-9_-]{10,}\\.[a-zA-Z0-9_-]{10,}\\.[a-zA-Z0-9_-]{10,})"), // JWT
        Pattern.compile("-----BEGIN (RSA |EC |DSA )?PRIVATE KEY-----")
    );

    public String filter(String content) {
        if (content == null) return null;
        String result = content;
        for (Pattern p : PATTERNS) {
            result = p.matcher(result).replaceAll("***REDACTED***");
        }
        return result;
    }
}
